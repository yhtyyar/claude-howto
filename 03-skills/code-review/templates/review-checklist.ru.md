# Чек-лист ревью кода

## Чек-лист безопасности
- [ ] Нет жёстко закодированных учётных данных или секретов
- [ ] Валидация входных данных на всех пользовательских вводах
- [ ] Защита от SQL-инъекций (параметризованные запросы)
- [ ] CSRF-защита на операциях изменения состояния
- [ ] Защита от XSS с правильным экранированием
- [ ] Проверки аутентификации на защищённых эндпоинтах
- [ ] Проверки авторизации на ресурсах
- [ ] Безопасное хэширование паролей (bcrypt, argon2)
- [ ] Нет конфиденциальных данных в логах
- [ ] HTTPS принудительно применён

## Чек-лист производительности
- [ ] Нет N+1 запросов
- [ ] Правильное использование индексов
- [ ] Кэширование реализовано там, где полезно
- [ ] Нет блокирующих операций в основном потоке
- [ ] Async/await используется корректно
- [ ] Большие наборы данных разбиты на страницы
- [ ] Пул соединений с базой данных
- [ ] Оптимизированные регулярные выражения
- [ ] Нет ненужного создания объектов
- [ ] Утечки памяти предотвращены

## Чек-лист качества
- [ ] Функции < 50 строк
- [ ] Понятное именование переменных
- [ ] Нет дублированного кода
- [ ] Правильная обработка ошибок
- [ ] Комментарии объясняют ПОЧЕМУ, а не ЧТО
- [ ] Нет console.log в production
- [ ] Проверка типов (TypeScript/JSDoc)
- [ ] Принципы SOLID соблюдены
- [ ] Паттерны проектирования применены корректно
- [ ] Самодокументирующийся код

## Чек-лист тестирования
- [ ] Модульные тесты написаны
- [ ] Граничные случаи охвачены
- [ ] Сценарии ошибок протестированы
- [ ] Интеграционные тесты присутствуют
- [ ] Покрытие > 80%
- [ ] Нет нестабильных тестов
- [ ] Внешние зависимости замоканы
- [ ] Чёткие названия тестов
