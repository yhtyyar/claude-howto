## Проблема

Ты установил Claude Code. Запустил несколько промптов. И что дальше?

- **Официальная документация описывает возможности — но не показывает, как их сочетать.** Ты знаешь, что слэш-команды существуют, но не знаешь, как объединить их с хуками, памятью и субагентами в рабочий процесс, который реально экономит часы времени.
- **Нет чёткого учебного маршрута.** Учить MCP до хуков? Навыки до субагентов? В итоге ты просматриваешь всё подряд и так ничего не осваиваешь.
- **Примеры слишком примитивны.** «Hello World»-команда не поможет тебе построить production-пайплайн для код-ревью с памятью, делегированием специализированным агентам и автоматическим сканированием безопасности.

Ты используешь лишь 10% возможностей Claude Code — и даже не знаешь, чего тебе не хватает.

## Как Claude How To решает её

Это не очередной справочник возможностей. Это **структурированное, визуальное, практическое руководство**, которое учит использовать каждую функцию Claude Code с реальными шаблонами, которые можно скопировать в свой проект прямо сейчас.

| | Официальная документация | Это руководство |
|--|--------------------------|-----------------|
| **Формат** | Справочная документация | Визуальные уроки с диаграммами Mermaid |
| **Глубина** | Описание возможностей | Как работает под капотом |
| **Примеры** | Базовые фрагменты | Production-готовые шаблоны для немедленного использования |
| **Структура** | Организация по функциям | Прогрессивный учебный маршрут (от новичка до эксперта) |
| **Онбординг** | Самостоятельный | Управляемый маршрут с оценкой времени |
| **Самооценка** | Отсутствует | Интерактивные тесты для выявления пробелов и построения персонального маршрута |

### Что тебя ждёт:

- **10 учебных модулей**, охватывающих все возможности Claude Code — от слэш-команд до команд специализированных агентов
- **Конфиги для копирования** — слэш-команды, шаблоны CLAUDE.md, скрипты хуков, конфиги MCP, определения субагентов и полные плагин-бандлы
- **Диаграммы Mermaid**, показывающие как работает каждая функция изнутри — чтобы ты понимал *почему*, а не просто *как*
- **Управляемый учебный маршрут** от новичка до продвинутого пользователя за 11–13 часов
- **Встроенная самооценка** — запусти `/self-assessment` или `/lesson-quiz hooks` прямо в Claude Code, чтобы выявить пробелы

## Как это работает

### 1. Определи свой уровень

Пройди [тест самооценки](LEARNING-ROADMAP.md#-find-your-level) или запусти `/self-assessment` в Claude Code. Получи персональный маршрут на основе того, что ты уже знаешь.

### 2. Следуй управляемому маршруту

Проходи 10 модулей по порядку — каждый строится на предыдущем. Копируй шаблоны прямо в свой проект по мере обучения.

### 3. Объединяй возможности в рабочие процессы

Настоящая сила — в сочетании функций. Учись соединять слэш-команды + память + субагенты + хуки в автоматизированные пайплайны для код-ревью, деплоя и генерации документации.

### 4. Проверяй своё понимание

Запускай `/lesson-quiz [тема]` после каждого модуля. Тест выявляет, что ты пропустил, чтобы быстро закрыть пробелы.

## Начать за 15 минут

```bash
# 1. Клонируй руководство
git clone https://github.com/luongnv89/claude-howto.git
cd claude-howto

# 2. Скопируй первую слэш-команду
mkdir -p /путь/к/твоему-проекту/.claude/commands
cp 01-slash-commands/optimize.md /путь/к/твоему-проекту/.claude/commands/

# 3. Попробуй — в Claude Code введи:
# /optimize

# 4. Готов к большему? Настрой память проекта:
cp 02-memory/project-CLAUDE.md /путь/к/твоему-проекту/CLAUDE.md

# 5. Установи навык:
cp -r 03-skills/code-review ~/.claude/skills/
```

Хочешь полную настройку? Вот **базовая настройка за 1 час**:

```bash
# Слэш-команды (15 мин)
cp 01-slash-commands/*.md .claude/commands/

# Память проекта (15 мин)
cp 02-memory/project-CLAUDE.md ./CLAUDE.md

# Установка навыка (15 мин)
cp -r 03-skills/code-review ~/.claude/skills/

# Цель на выходные: добавить хуки, субагенты, MCP и плагины
# Следуй учебному маршруту для пошаговой настройки
```

## Часто задаваемые вопросы

**Это бесплатно?**
Да. Лицензия MIT, бесплатно навсегда. Используй в личных проектах, на работе, в команде — без ограничений, кроме включения уведомления о лицензии.

**Поддерживается ли это руководство?**
Активно. Руководство синхронизируется с каждым релизом Claude Code. Текущая версия: v2.2.0 (март 2026), совместима с Claude Code 2.1+.

**Чем это отличается от официальной документации?**
Официальная документация — это справочник возможностей. Это руководство — урок с диаграммами, production-готовыми шаблонами и прогрессивным учебным маршрутом. Они дополняют друг друга — начни здесь, чтобы научиться, обращайся к официальной документации для деталей.

**Сколько времени занимает полный курс?**
11–13 часов для полного маршрута. Но ценность получишь уже через 15 минут — просто скопируй шаблон слэш-команды и попробуй.

**Работает ли это с Claude Sonnet / Haiku / Opus?**
Да. Все шаблоны работают с Claude Sonnet 4.6, Claude Opus 4.6 и Claude Haiku 4.5.

**Могу ли я внести вклад?**
Конечно. Смотри [CONTRIBUTING.md](CONTRIBUTING.md) с подробными инструкциями. Приветствуются новые примеры, исправления ошибок, улучшения документации и шаблоны сообщества.

**Могу ли я читать это офлайн?**
Да. Запусти `uv run scripts/build_epub.py` для создания EPUB-книги со всем контентом и отрисованными диаграммами.

<details>
<summary>06. Хуки</summary>

**Расположение**: [06-hooks/](06-hooks/)

**Что это**: Управляемые событиями shell-команды, которые автоматически выполняются в ответ на события Claude Code

**Примеры**:
- `format-code.sh` — Авто-форматирование кода перед записью
- `pre-commit.sh` — Запуск тестов перед коммитами
- `security-scan.sh` — Сканирование на уязвимости безопасности
- `log-bash.sh` — Логирование всех bash-команд
- `validate-prompt.sh` — Валидация пользовательских промптов

**Установка**:
```bash
mkdir -p ~/.claude/hooks
cp 06-hooks/*.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/*.sh
```

**Ключевые концепции**:

- **Перемотка**: Возврат к предыдущей контрольной точке
- **Точка ветвления**: Исследование нескольких подходов из одной контрольной точки

**Использование**:
```

/rewind

# Затем выбери из пяти вариантов:
# 1. Восстановить код и разговор
# 2. Восстановить разговор
# 3. Восстановить код
# 4. Резюмировать отсюда
# 5. Отмена
```

- **Режим планирования** — Создание детальных планов реализации перед кодированием

- **Управление сессиями** — `/resume`, `/rename`, `/fork`, `claude -c`, `claude -r`
- **Конфигурация** — Настройка поведения в `~/.claude/settings.json`

<details>
<summary>10. CLI-справочник</summary>

**Расположение**: [10-cli/](10-cli/)

**Что это**: Полный справочник интерфейса командной строки для Claude Code

**Быстрые примеры**:
```bash
# Интерактивный режим
claude "объясни этот проект"

# Режим печати (неинтерактивный)
claude -p "проверь этот код"

# Обработка содержимого файла
cat error.log | claude -p "объясни эту ошибку"

# JSON-вывод для скриптов
claude -p --output-format json "перечисли функции"

# Возобновить сессию
claude -r "feature-auth" "продолжи реализацию"
```

**Сценарии использования**: Интеграция с CI/CD-пайплайном, автоматизация скриптов, пакетная обработка, многосессионные рабочие процессы, кастомные конфигурации агентов

</details>

<details>
<summary>Примеры рабочих процессов</summary>

### Полный рабочий процесс код-ревью

```markdown
# Использует: Слэш-команды + Субагенты + Память + MCP

Пользователь: /review-pr

Claude:
1. Загружает память проекта (стандарты кодирования)
2. Получает PR через GitHub MCP
3. Делегирует субагенту code-reviewer
4. Делегирует субагенту test-engineer
5. Синтезирует результаты
6. Предоставляет комплексный ревью
```

### Автоматизированная документация

```markdown
# Использует: Навыки + Субагенты + Память

Пользователь: "Создай API-документацию для модуля авторизации"

Claude:
1. Загружает память проекта (стандарты документации)
2. Обнаруживает запрос генерации документации
3. Авто-вызывает навык doc-generator
4. Делегирует субагенту api-documenter
5. Создаёт комплексную документацию с примерами
```

### DevOps-деплой

```markdown
# Использует: Плагины + MCP + Хуки

Пользователь: /deploy production

Claude:
1. Запускает pre-deploy хук (валидирует окружение)
2. Делегирует субагенту deployment-specialist
3. Выполняет деплой через Kubernetes MCP
4. Мониторит прогресс
5. Запускает post-deploy хук (проверки работоспособности)
6. Сообщает статус
```

</details>

<details>
<summary>Лучшие практики</summary>

### Делай
- Начинай с простых слэш-команд
- Добавляй функции постепенно
- Используй память для стандартов команды
- Тестируй конфигурации локально сначала
- Документируй кастомные реализации
- Держи конфигурации проекта под контролем версий
- Делись плагинами с командой

### Не делай
- Не создавай избыточные функции
- Не хардкоди учётные данные
- Не пропускай документацию
- Не усложняй простые задачи
- Не игнорируй лучшие практики безопасности
- Не коммить чувствительные данные

<details>
<summary>Устранение неполадок</summary>

### Функция не загружается
1. Проверь расположение и именование файла
2. Проверь синтаксис YAML frontmatter
3. Проверь права доступа к файлу
4. Проверь совместимость с версией Claude Code

### Соединение MCP не удалось
1. Проверь переменные окружения
2. Проверь установку MCP-сервера
3. Протестируй учётные данные
4. Проверь сетевое подключение

### Субагент не делегирует
1. Проверь права инструментов
2. Проверь чёткость описания агента
3. Проверь сложность задачи
4. Протестируй агента самостоятельно

</details>
