## Обзор

Расширенные функции Claude Code дополняют основные возможности механизмами планирования, рассуждений, автоматизации и контроля. Эти функции обеспечивают сложные рабочие процессы для комплексных задач разработки, код-ревью, автоматизации и управления несколькими сессиями.

**Ключевые расширенные функции:**
- **Режим планирования**: Создание подробных планов реализации перед кодированием
- **Расширенное мышление**: Глубокое рассуждение для сложных проблем
- **Автоматический режим**: Фоновый классификатор безопасности проверяет каждое действие (Research Preview)
- **Фоновые задачи**: Запуск длительных операций без блокировки разговора
- **Режимы разрешений**: Управление действиями Claude (`default`, `acceptEdits`, `plan`, `auto`, `dontAsk`, `bypassPermissions`)
- **Print Mode**: Запуск Claude Code в неинтерактивном режиме для автоматизации и CI/CD (`claude -p`)
- **Управление сессиями**: Управление несколькими рабочими сессиями
- **Голосовой ввод**: Push-to-talk голосовой ввод с поддержкой STT на 20 языках
- **Каналы**: MCP-серверы отправляют сообщения в запущенные сессии (Research Preview)
- **Удалённое управление**: Управление Claude Code с claude.ai или приложения Claude
- **Веб-сессии**: Запуск Claude Code в браузере на claude.ai/code
- **Десктопное приложение**: Автономное приложение для просмотра diff и нескольких сессий

## Режим планирования

Режим планирования позволяет Claude обдумать сложные задачи перед их реализацией, создавая подробный план для проверки и одобрения.

### Когда использовать режим планирования

✅ Использовать при:
- Сложном многофайловом рефакторинге
- Реализации новых функций
- Архитектурных изменениях
- Миграциях баз данных
- Крупных редизайнах API

❌ Не использовать при:
- Простых исправлениях ошибок
- Изменениях форматирования
- Правке одного файла
- Быстрых запросах

### Активация режима планирования

**Слэш-команда**:
```bash
/plan Реализовать систему аутентификации пользователей
```

**Флаг CLI**:
```bash
claude --permission-mode plan
```

**Горячая клавиша**:
- `Shift + Tab` — Переключение режимов разрешений (включая plan)
- `Alt + M` — Переключение режимов (Windows/Linux)

**Псевдоним модели для планирования**: Используй `opusplan` для использования Opus при планировании и Sonnet для выполнения:

```bash
claude --model opusplan "спроектируй и реализуй новый API"
```

**Редактировать план внешне**: Нажми `Ctrl+G` для открытия текущего плана во внешнем редакторе.

### Пример: Реализация функции

```
Пользователь: /plan Добавить систему уведомлений в реальном времени

Claude: Создам план для реализации уведомлений в реальном времени.

## План реализации

### Фаза 1: Настройка инфраструктуры
1. Установить библиотеки WebSocket (socket.io)
2. Настроить WebSocket-сервер
3. Настроить управление соединениями

### Фаза 2: Серверная реализация
4. Создать сервис уведомлений
5. Реализовать очередь уведомлений
6. Добавить схему БД для уведомлений

### Фаза 3: Клиентская интеграция
7. Настроить WebSocket-клиент
8. Создать UI-компоненты уведомлений

### Фаза 4: Тестирование
9. Написать модульные тесты
10. Написать интеграционные тесты

Предполагаемое время: 3-4 часа | Файлов для изменения: 12 | Новых файлов: 8

Готов продолжить? (да/нет/изменить план)
```

## Расширенное мышление

Расширенное мышление позволяет Claude тратить больше времени на рассуждения о сложных проблемах.

### Активация расширенного мышления

**Горячая клавиша**:
- `Option + T` (macOS) / `Alt + T` (Windows/Linux) — Переключить расширенное мышление

**Автоматическая активация**:
- Включена по умолчанию для всех моделей (Opus 4.6, Sonnet 4.6, Haiku 4.5)
- Opus 4.6: Адаптивные рассуждения с уровнями усилий: `low` (○), `medium` (◐), `high` (●), `max` (только Opus 4.6)

**Кастомный бюджет**:
```bash
export MAX_THINKING_TOKENS=1024
```

**Уровень усилий** (только Opus 4.6):
```bash
export CLAUDE_CODE_EFFORT_LEVEL=high   # low (○), medium (◐), high (●), или max
```

**Флаг CLI**:
```bash
claude --effort high "сложный архитектурный анализ"
```

**Слэш-команда**:
```
/effort high
```

> **Примечание:** Ключевое слово «ultrathink» в промптах активирует режим глубокого рассуждения.

### Пример: Архитектурное решение

```
Пользователь: /think Стоит ли нам использовать микросервисы или монолит для нашей платформы?

Claude: Позволь тщательно обдумать это архитектурное решение...

<extended_thinking>
Анализ:
- Размер команды: 5 разработчиков
- Ожидаемый трафик: 10k пользователей в день
- Потребность в масштабировании до 100k+ пользователей

Рекомендация: Начать с модульного монолита:
✓ Быстрая начальная разработка
✓ Низкая операционная сложность
✓ Возможность последующего извлечения сервисов
</extended_thinking>

Рекомендация: Реализовать модульный монолит с чёткими границами модулей.
```

## Автоматический режим

Автоматический режим — это режим разрешений Research Preview (март 2026), использующий фоновый классификатор безопасности для проверки каждого действия перед выполнением.

### Требования

- **Тарифный план**: Team (Enterprise и API внедряются)
- **Модель**: Claude Sonnet 4.6 или Opus 4.6

### Включение автоматического режима

```bash
# Разблокировать авторежим с флагом CLI
claude --enable-auto-mode

# Или установить как режим разрешений по умолчанию
claude --permission-mode auto
```

### Действия, блокируемые по умолчанию

| Заблокированное действие | Пример |
|-------------------------|--------|
| Установка через pipe-to-shell | `curl \| bash` |
| Отправка чувствительных данных наружу | API-ключи, учётные данные по сети |
| Деплой в production | Команды деплоя на production |
| Массовое удаление | `rm -rf` для больших директорий |
| Изменения IAM | Изменения разрешений и ролей |
| Force push в main | `git push --force origin main` |

### Действия, разрешённые по умолчанию

| Разрешённое действие | Пример |
|---------------------|--------|
| Локальные операции с файлами | Чтение, запись, редактирование файлов проекта |
| Установка объявленных зависимостей | `npm install`, `pip install` из манифеста |
| HTTP только для чтения | `curl` для получения документации |
| Push в текущую ветку | `git push origin feature-branch` |

## Фоновые задачи

Фоновые задачи позволяют длительным операциям выполняться без блокировки разговора.

### Базовое использование

```bash
Пользователь: Запусти тесты в фоновом режиме

Claude: Задача bg-1234 запущена

/task list           # Показать все задачи
/task status bg-1234 # Проверить прогресс
/task show bg-1234   # Просмотр вывода
/task cancel bg-1234 # Отменить задачу
```

### Пример: Параллельная разработка

```
Пользователь: Запусти сборку в фоновом режиме
Claude: Запускаю сборку... (task-id: bg-5001)

Пользователь: Также запусти линтер в фоне
Claude: Запускаю линтер... (task-id: bg-5002)

Пользователь: Пока они работают, реализуем новый API-эндпоинт
Claude: [Реализует эндпоинт пока сборка и линтер работают в фоне]

[10 минут спустя]
Claude: 📢 Сборка завершена успешно (bg-5001)
📢 Линтер нашёл 12 проблем (bg-5002)
```

## Запланированные задачи

Запланированные задачи позволяют автоматически запускать промпты по расписанию или как одноразовые напоминания. Доступны с v2.1.72+.

### Команда `/loop`

```bash
# Явный интервал
/loop 5m проверь, завершился ли деплой

# На естественном языке
/loop проверяй статус сборки каждые 30 минут
```

Поддерживаются стандартные 5-полевые cron-выражения.

### Одноразовые напоминания

```
напомни мне в 15:00 запушить ветку с релизом
через 45 минут запусти интеграционные тесты
```

### Управление запланированными задачами

| Инструмент | Описание |
|-----------|---------|
| `CronCreate` | Создать новую запланированную задачу |
| `CronList` | Список всех активных запланированных задач |
| `CronDelete` | Удалить запланированную задачу |

**Ограничения:**
- До **50 запланированных задач** за сессию
- Привязаны к сессии — очищаются при завершении сессии
- Повторяющиеся задачи автоматически истекают через **3 дня**

## Режимы разрешений

Режимы разрешений управляют тем, какие действия Claude может выполнять без явного одобрения.

### Доступные режимы разрешений

| Режим | Поведение |
|-------|----------|
| `default` | Только чтение файлов; запрашивает все остальные действия |
| `acceptEdits` | Чтение и редактирование файлов; запрашивает команды |
| `plan` | Только чтение файлов (исследовательский режим, без правок) |
| `auto` | Все действия с проверками фонового классификатора безопасности (Research Preview) |
| `bypassPermissions` | Все действия, без проверок разрешений (опасно) |
| `dontAsk` | Выполняются только предварительно одобренные инструменты |

Переключай режимы с помощью `Shift+Tab` в CLI. Установи режим по умолчанию с флагом `--permission-mode`.

### Примеры режимов разрешений

#### Режим Default
```
Пользователь: Исправь баг в auth.ts
Claude: Мне нужно изменить src/auth.ts для исправления бага.
Одобрить это изменение? (да/нет/показать)
```

#### Режим Plan
```
Пользователь: /plan Реализовать систему аутентификации
Claude: Создам план реализации аутентификации.
## План реализации
[Подробный план]
Готов продолжить? (да/нет/изменить)
```

## Безголовый режим

Print Mode (`claude -p`) позволяет Claude Code работать без интерактивного ввода — идеально для автоматизации и CI/CD.

### Запуск в Print Mode

```bash
# Запустить конкретную задачу
claude -p "Запустить все тесты"

# Обработать перенаправленный контент
cat error.log | claude -p "Проанализировать эти ошибки"
```

### Пример: Интеграция CI/CD (GitHub Actions)

```yaml
# .github/workflows/code-review.yml
name: AI Code Review

on: [pull_request]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Claude Code Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p --output-format json \
            --max-turns 3 \
            "Проверь этот PR на наличие проблем безопасности и качества кода" > review.json
```

## Управление сессиями

### Команды управления сессиями

| Команда | Описание |
|---------|---------|
| `/resume` | Возобновить разговор по ID или имени |
| `/rename` | Назвать текущую сессию |
| `/fork` | Создать ответвление текущей сессии |
| `claude -c` | Продолжить последний разговор |
| `claude -r "session"` | Возобновить сессию по имени или ID |

### Примеры

```bash
# Продолжить последний разговор
claude -c

# Возобновить именованную сессию
claude -r "auth-refactor" "завершить этот PR"

# Создать ответвление для другого подхода
claude --resume auth-refactor --fork-session "попробовать OAuth вместо"
```

## Интерактивные функции

### Горячие клавиши

| Клавиша | Описание |
|--------|---------|
| `Ctrl+C` | Отменить текущий ввод/генерацию |
| `Ctrl+D` | Выйти из Claude Code |
| `Ctrl+G` | Редактировать план во внешнем редакторе |
| `Ctrl+L` | Очистить экран терминала |
| `Ctrl+O` | Переключить подробный вывод (просмотр рассуждений) |
| `Ctrl+R` | Обратный поиск по истории |
| `Ctrl+T` | Переключить вид списка задач |
| `Ctrl+B` | Перевести запущенные задачи в фоновый режим |
| `Esc+Esc` | Откат кода/разговора |
| `Shift+Tab` / `Alt+M` | Переключить режимы разрешений |
| `Option+P` / `Alt+P` | Переключить модель |
| `Option+T` / `Alt+T` | Переключить расширенное мышление |

### Кастомизация горячих клавиш

Создавай кастомные горячие клавиши через `/keybindings` (v2.1.18+):

```json
{
  "$schema": "https://www.schemastore.org/claude-code-keybindings.json",
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "ctrl+e": "chat:externalEditor",
        "ctrl+u": null
      }
    }
  ]
}
```

## Голосовой ввод

Голосовой ввод предоставляет push-to-talk функциональность с поддержкой STT на 20 языках.

**Включение голосового ввода**:
```bash
# Держи клавишу активации и говори
```

**Настройка**:
```json
{
  "voice": {
    "enabled": true,
    "language": "ru-RU",
    "pushToTalkKey": "Alt"
  }
}
```

## Конфигурация и настройки

### Файлы настроек

| Файл | Область | Описание |
|------|---------|---------|
| `~/.claude/settings.json` | Пользователь | Глобальные пользовательские настройки |
| `.claude/settings.json` | Проект | Настройки проекта, разделяемые с командой |
| `.claude/settings.local.json` | Локальный | Локальные настройки (не в git) |

### Общие настройки

```json
{
  "model": "sonnet",
  "permissions": {
    "defaultMode": "default"
  },
  "verbose": false,
  "maxTurns": 10,
  "theme": "dark"
}
```

### Команды конфигурации

```bash
/config          # Открыть интерактивное меню конфигурации
/theme           # Сменить тему (dark/light/auto)
/fast            # Переключить быстрый режим
/verbose         # Переключить подробный режим
```

## Лучшие практики

| Сценарий | Рекомендуемые настройки |
|---------|----------------------|
| Разработка функций | `plan` режим для сложных изменений |
| Код-ревью | `plan` режим (только чтение) |
| Автоматизация CI/CD | Print mode с `--max-turns` |
| Парное программирование | `default` режим с одобрением |
| Быстрые исправления | `acceptEdits` для скорости |
| Исследование кодовой базы | `plan` или субагент Explore |
| Длительные задачи | Фоновые задачи |
| Сложные проблемы | Расширенное мышление с `--effort high` |
