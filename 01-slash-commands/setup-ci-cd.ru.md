# Настройка CI/CD пайплайна

Реализовать комплексные DevOps-ворота качества с учётом типа проекта:

1. **Анализировать проект**: определить язык(и), фреймворк, систему сборки и существующий инструментарий
2. **Настроить pre-commit хуки** с языково-специфическими инструментами:
   - Форматирование: Prettier/Black/gofmt/rustfmt/и др.
   - Линтинг: ESLint/Ruff/golangci-lint/Clippy/и др.
   - Безопасность: Bandit/gosec/cargo-audit/npm audit/и др.
   - Проверка типов: TypeScript/mypy/flow (если применимо)
   - Тесты: запустить соответствующие тестовые наборы
3. **Создать GitHub Actions рабочие процессы** (.github/workflows/):
   - Продублировать pre-commit проверки при push/PR
   - Матрица нескольких версий/платформ (если применимо)
   - Верификация сборки и тестирования
   - Шаги развёртывания (если необходимо)
4. **Проверить пайплайн**: протестировать локально, создать тестовый PR, убедиться в прохождении всех проверок

Использовать бесплатные/открытые инструменты. Соблюдать существующие конфигурации. Поддерживать быстрое выполнение.
