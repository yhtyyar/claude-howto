## Контекст

- Текущий статус git: !`git status`
- Текущий git diff: !`git diff HEAD`
- Текущая ветка: !`git branch --show-current`
- Недавние коммиты: !`git log --oneline -10`

## Ваша задача

На основе вышеуказанных изменений создайте единый git-коммит.

Если сообщение было передано через аргументы, используйте его: $ARGUMENTS

В противном случае проанализируйте изменения и создайте подходящее сообщение коммита в формате conventional commits:
- `feat:` для новых функций
- `fix:` для исправлений ошибок
- `docs:` для изменений документации
- `refactor:` для рефакторинга кода
- `test:` для добавления тестов
- `chore:` для задач обслуживания
