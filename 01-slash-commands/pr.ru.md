# Чек-лист подготовки Pull Request

Перед созданием PR выполните следующие шаги:

1. Запустить линтер: `prettier --write .`
2. Запустить тесты: `npm test`
3. Проверить git diff: `git diff HEAD`
4. Зафиксировать изменения: `git add .`
5. Создать сообщение коммита в формате conventional commits:
   - `fix:` для исправлений ошибок
   - `feat:` для новых функций
   - `docs:` для документации
   - `refactor:` для реструктуризации кода
   - `test:` для добавления тестов
   - `chore:` для обслуживания

6. Сформировать сводку PR, включая:
   - Что изменилось
   - Почему изменилось
   - Проведённое тестирование
   - Возможные последствия
