# Расширение модульных тестов

Расширьте существующие модульные тесты с учётом тестового фреймворка проекта:

1. **Анализировать покрытие**: запустить отчёт покрытия для определения непротестированных ветвей, граничных случаев и областей с низким покрытием
2. **Определить пробелы**: проверить код на логические ветки, пути ошибок, граничные условия, пустые/null входные данные
3. **Написать тесты** с использованием фреймворка проекта:
   - Jest/Vitest/Mocha (JavaScript/TypeScript)
   - pytest/unittest (Python)
   - Go testing/testify (Go)
   - Rust test framework (Rust)
4. **Ориентироваться на конкретные сценарии**:
   - Обработка ошибок и исключений
   - Граничные значения (мин/макс, пустые, null)
   - Граничные случаи и крайние ситуации
   - Переходы состояний и побочные эффекты
5. **Проверить улучшение**: повторно запустить покрытие, убедиться в измеримом увеличении

Показывать только блоки нового тестового кода. Следовать существующим шаблонам тестов и соглашениям именования.
