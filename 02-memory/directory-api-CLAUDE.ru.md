# Стандарты API-модуля

Этот файл переопределяет корневой CLAUDE.md для всего в /src/api/

## API-специфичные стандарты

### Валидация запросов
- Использовать Zod для валидации схем
- Всегда валидировать входные данные
- Возвращать 400 с ошибками валидации
- Включать детали ошибок на уровне поля

### Аутентификация
- Все эндпоинты требуют JWT-токен
- Токен в заголовке Authorization
- Токен истекает через 24 часа
- Реализовать механизм refresh-токена

### Формат ответа

Все ответы должны следовать этой структуре:

```json
{
  "success": true,
  "data": { /* фактические данные */ },
  "timestamp": "2025-11-06T10:30:00Z",
  "version": "1.0"
}
```

Ответы с ошибками:
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Сообщение пользователю",
    "details": { /* ошибки полей */ }
  },
  "timestamp": "2025-11-06T10:30:00Z"
}
```

### Пагинация
- Использовать курсорную пагинацию (не по смещению)
- Включать булево `hasMore`
- Ограничить максимальный размер страницы до 100
- Размер страницы по умолчанию: 20

### Ограничение запросов
- 1000 запросов в час для аутентифицированных пользователей
- 100 запросов в час для публичных эндпоинтов
- Возвращать 429 при превышении
- Включать заголовок retry-after

### Кэширование
- Использовать Redis для кэширования сессий
- Длительность кэша: по умолчанию 5 минут
- Инвалидировать при операциях записи
- Тегировать ключи кэша типом ресурса
