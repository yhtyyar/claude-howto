# Мои предпочтения в разработке

## О себе
- **Уровень опыта**: 8 лет full-stack разработки
- **Предпочитаемые языки**: TypeScript, Python
- **Стиль общения**: Прямой, с примерами
- **Стиль обучения**: Визуальные диаграммы с кодом

## Предпочтения в коде

### Обработка ошибок
Предпочитаю явную обработку ошибок с блоками try-catch и содержательными сообщениями об ошибках.
Избегать общих ошибок. Всегда логировать ошибки для отладки.

### Комментарии
Использовать комментарии для ПОЧЕМУ, не ЧТО. Код должен быть самодокументирующимся.
Комментарии должны объяснять бизнес-логику или неочевидные решения.

### Тестирование
Предпочитаю TDD (разработка через тестирование).
Сначала писать тесты, затем реализацию.
Фокус на поведении, не на деталях реализации.

### Архитектура
Предпочитаю модульный, слабосвязанный дизайн.
Использовать внедрение зависимостей для тестируемости.
Разделение ответственностей (Контроллеры, Сервисы, Репозитории).

## Предпочтения в отладке
- Использовать console.log с префиксом: `[DEBUG]`
- Включать контекст: имя функции, релевантные переменные
- Использовать стек вызовов при наличии
- Всегда включать временные метки в логах

## Общение
- Объяснять сложные концепции с диаграммами
- Показывать конкретные примеры перед объяснением теории
- Включать фрагменты кода до/после
- Резюмировать ключевые моменты в конце

## Организация проекта
Организую проекты следующим образом:
```
project/
  ├── src/
  │   ├── api/
  │   ├── services/
  │   ├── models/
  │   └── utils/
  ├── tests/
  ├── docs/
  └── docker/
```

## Инструментарий
- **IDE**: VS Code с vim-биндингами
- **Терминал**: Zsh с Oh-My-Zsh
- **Форматирование**: Prettier (длина строки 100 символов)
- **Линтер**: ESLint с конфигурацией airbnb
- **Тестовый фреймворк**: Jest с React Testing Library
