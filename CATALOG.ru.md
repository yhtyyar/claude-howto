## Режимы разрешений

Claude Code поддерживает 6 режимов разрешений, управляющих авторизацией использования инструментов.

| Режим | Описание | Когда использовать |
|-------|---------|-------------------|
| `default` | Запрашивать каждый вызов инструмента | Стандартное интерактивное использование |
| `acceptEdits` | Автопринятие правок файлов, запрос для остального | Доверенные рабочие процессы редактирования |
| `plan` | Только инструменты чтения, без записи | Планирование и исследование |
| `auto` | Принять все инструменты без запросов | Полностью автономная работа (Research Preview) |
| `bypassPermissions` | Пропустить все проверки разрешений | CI/CD, безголовые среды |
| `dontAsk` | Пропустить инструменты, требующие разрешения | Неинтерактивный скриптинг |

> **Примечание**: Режим `auto` — функция Research Preview (март 2026). Используйте `bypassPermissions` только в доверенных изолированных средах.

### Распространённые MCP-серверы

| Сервер | Описание | Когда использовать | Область | Установка |
|--------|---------|-------------------|---------|-----------|
| **GitHub** | Управление PR, issues, код | GitHub-рабочие процессы | Проект | `claude mcp add github -- npx -y @modelcontextprotocol/server-github` |
| **Database** | SQL-запросы, доступ к данным | Операции с базой данных | Проект | `claude mcp add db -- npx -y @modelcontextprotocol/server-postgres` |
| **Filesystem** | Расширенные файловые операции | Сложные задачи с файлами | Пользователь | `claude mcp add fs -- npx -y @modelcontextprotocol/server-filesystem` |
| **Slack** | Командная коммуникация | Уведомления, обновления | Проект | Настроить в settings |
| **Google Docs** | Доступ к документам | Редактирование документов | Проект | Настроить в settings |
| **Memory** | Постоянная память | Воспроизведение между сессиями | Пользователь | Настроить в settings |

### Пример конфигурации MCP

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

**Быстрая установка GitHub MCP**:
```bash
export GITHUB_TOKEN="твой_токен" && claude mcp add github -- npx -y @modelcontextprotocol/server-github
```

## Хуки

Событийная автоматизация, выполняющая shell-команды при событиях Claude Code.

### События хуков

| Событие | Описание | Когда срабатывает | Сценарии использования |
|---------|---------|-----------------|----------------------|
| `SessionStart` | Начало/возобновление сессии | Инициализация сессии | Задачи настройки |
| `InstructionsLoaded` | Загружены инструкции | CLAUDE.md или файл правил загружен | Кастомная обработка инструкций |
| `UserPromptSubmit` | До обработки промпта | Пользователь отправляет сообщение | Валидация ввода |
| `PreToolUse` | До выполнения инструмента | Перед запуском любого инструмента | Валидация, логирование |
| `PermissionRequest` | Показан диалог разрешений | Перед чувствительными действиями | Кастомные потоки одобрения |
| `PostToolUse` | После успешного инструмента | После завершения любого инструмента | Форматирование, уведомления |
| `PostToolUseFailure` | Выполнение инструмента не удалось | После ошибки инструмента | Обработка ошибок, логирование |
| `SubagentStart` | Субагент порождён | Задача субагента начинается | Инициализация контекста |
| `SubagentStop` | Субагент завершился | Задача субагента выполнена | Цепочка действий |
| `Stop` | Claude заканчивает ответ | Ответ завершён | Очистка, отчётность |
| `TeammateIdle` | Агент-тиммейт простаивает | Координация команды агентов | Распределение работы |
| `TaskCompleted` | Задача помечена как выполненная | Задача выполнена | Обработка после задачи |
| `ConfigChange` | Конфигурация обновлена | Настройки изменены | Реагирование на изменения |
| `CwdChanged` | Рабочая директория изменилась | Директория изменена | Настройка для конкретной директории |
| `FileChanged` | Отслеживаемый файл изменился | Файл изменён | Мониторинг файлов, пересборка |
| `PreCompact` | До операции компактизации | Сжатие контекста | Сохранение состояния |
| `PostCompact` | После завершения компактизации | Компактизация завершена | Действия после компактизации |
| `WorktreeCreate` | Создаётся worktree | Git worktree создан | Настройка среды worktree |
| `WorktreeRemove` | Удаляется worktree | Git worktree удалён | Очистка ресурсов |
| `Elicitation` | MCP-сервер запрашивает ввод | MCP elicitation | Валидация ввода |
| `ElicitationResult` | Пользователь ответил на запрос | Пользователь отвечает | Обработка ответа |
| `SessionEnd` | Сессия завершается | Завершение сессии | Очистка, сохранение состояния |

### Примеры хуков

| Хук | Описание | Событие | Область | Установка |
|-----|---------|--------|---------|-----------|
| `validate-bash.py` | Валидация команд | PreToolUse:Bash | Проект | `cp 06-hooks/validate-bash.py .claude/hooks/` |
| `security-scan.py` | Сканирование безопасности | PostToolUse:Write | Проект | `cp 06-hooks/security-scan.py .claude/hooks/` |
| `format-code.sh` | Авто-форматирование | PostToolUse:Write | Пользователь | `cp 06-hooks/format-code.sh ~/.claude/hooks/` |
| `validate-prompt.py` | Валидация промптов | UserPromptSubmit | Проект | `cp 06-hooks/validate-prompt.py .claude/hooks/` |
| `context-tracker.py` | Отслеживание использования токенов | Stop | Пользователь | `cp 06-hooks/context-tracker.py ~/.claude/hooks/` |
| `pre-commit.sh` | Валидация перед коммитом | PreToolUse:Bash | Проект | `cp 06-hooks/pre-commit.sh .claude/hooks/` |
| `log-bash.sh` | Логирование команд | PostToolUse:Bash | Пользователь | `cp 06-hooks/log-bash.sh ~/.claude/hooks/` |

### Конфигурация хука

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "~/.claude/hooks/validate-bash.py"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "command": "~/.claude/hooks/format-code.sh"
      }
    ]
  }
}
```

**Быстрая установка всех хуков**:
```bash
mkdir -p ~/.claude/hooks && cp 06-hooks/*.sh ~/.claude/hooks/ && chmod +x ~/.claude/hooks/*.sh
```

## Файлы памяти

Постоянный контекст, загружаемый автоматически между сессиями.

### Типы памяти

| Тип | Расположение | Область | Когда использовать |
|-----|-------------|---------|-------------------|
| **Managed Policy** | Политики организации | Организация | Применять стандарты организации |
| **Проект** | `./CLAUDE.md` | Проект (команда) | Командные стандарты, контекст проекта |
| **Правила проекта** | `.claude/rules/` | Проект (команда) | Модульные правила проекта |
| **Пользователь** | `~/.claude/CLAUDE.md` | Пользователь (личный) | Личные предпочтения |
| **Правила пользователя** | `~/.claude/rules/` | Пользователь (личный) | Модульные личные правила |
| **Локальный** | `./CLAUDE.local.md` | Локальный (git-ignored) | Машинно-специфичные переопределения |
| **Автопамять** | Автоматическая | Сессия | Авто-сохранённые инсайты и исправления |

cp 02-memory/project-CLAUDE.md ./CLAUDE.md
